    üìò Components in Angular

    • Components are the smallest UI building blocks in Angular applications.
    • They are reusable ‚Üí one component can be used in multiple places.
    • Each component has:
    • HTML Template ‚Üí Defines the view (UI).
    • CSS / SCSS Styles ‚Üí Defines look & feel.
    • TypeScript Class ‚Üí Defines logic (data + functions).

    ‚úÖ Example Usage

    <!-- Reusing components -->

    üéØ Types of Selectors

    Angular lets you define how components are used in HTML:

    1. Element Selector (Default)
    2. Declared as:
      ts selector: 'app-user'

    3. Used as:
      html <app-user></app-user>

    4. Attribute Selector

    5. Declared as:
      ts selector: '[app-name]'

    6. Used as:
      html <div app-name></div>

    7. Class Selector

    8. Declared as:
      ts selector: '.app-name'
    9. Used as:
      html <div class="app-name"></div>

    üé® Styling Components

    • Each component can have its own CSS/SCSS files.
    • By default, styles apply only to that component (not global).
    • This is possible because Angular uses View Encapsulation.

    üîí View Encapsulation in Angular

    Encapsulation = Controlling how CSS styles are applied to components. 

    import { Component, ViewEncapsulation } from '@angular/core';

@Component({
  selector: 'app-user',
  templateUrl: './user.component.html',
  styleUrls: ['./user.component.css'],
  encapsulation: ViewEncapsulation.Emulated // default
})
export class UserComponent {}

    üõ† Types of ViewEncapsulation

    1. Emulated (Default)
    2. Angular adds unique attributes to isolate styles.
    3. Prevents styles from leaking outside.
      css /* user.component.css */ h2 { color: blue; } ‚úÖ Only <h2> inside <app-user> becomes blue.
    1. None
    2. Styles are global ‚Üí affect all components.
    3. Not recommended (may cause conflicts).
      ts encapsulation: ViewEncapsulation.None ‚úÖ Example:
      css .btn { color: red; } ‚Üí All buttons in the entire app turn red.
    1. ShadowDom
    2. Uses the browser‚Äôs native Shadow DOM.
    3. Styles are isolated inside the component‚Äôs shadow root.
    4. Global styles do not apply here.
      ts encapsulation: ViewEncapsulation.ShadowDom

    üí° Real-life analogy:
    Think of Encapsulation as soundproof rooms in a house:
    - Emulated ‚Üí Your music stays in your room, no noise outside.
    - None ‚Üí Everyone in the house hears your music.
    - ShadowDom ‚Üí A special high-tech soundproofing (browser‚Äôs built-in).
